    Step 1: GitHub Student Developer Pack

    The GitHub student developer pack is a MUST for every college going student. It gives you access to premium packages of Canva, Microsoft Azure, JetBrains and 100s more. The one we'll be needing for making the blog is - Namecheap.

    Sign up for the student developer pack by entering your details along with a picture of your college ID for verification. That's it.
    Then head on to the benefits page and get student benefits. There you will find Namecheap's free .me domain name benefit. Link your Namecheap account with your GitHub account and get a GitHub repository with the domain name integrated.

    Step 3: WGET

    Here's the thing. We're going to host our website on GitHub for free. GitHub only allows us to host a static website i.e no backend features. What do we mean by that? Try using the search button on this website, it won't work because it's a non static feature. It's nonetheless there because it would work when you host it on Ghost.org let's say (but you'll have to pay the yearly fee).

    Static features are all you require as a student for a blog. At the end of the blog I've added the method to make a dynamic website for free as well, but I'd suggest you going for it if you're comfortable with static first.

    GitHub websites should be made of .html files, whereas our themes are .hbs files.
    An HBS file is a template file created by Handlebars, a web template system. It contains a template written in HTML code and embedded with Handlebars expressions. An HBS file performs the same function as a .HANDLEBARS file. We need to convert these into .html files before we can deploy them on GitHub.

    Wget = Web Get. It crawls the websites and gets the HTML files we need for our GitHub page.
    Download wget by following these instructions: How to Use wget Command With Examples | PhoenixNAP KB
    Now,
    1) Go to the Joelma/(your theme name folder) inside your editor and make a new folder inside it called docs. (very important)
    2) Using your editor, open the terminal INSIDE THE JOELMA/(your theme) FOLDER.
    Not inside docs. Not in C: drive. We need a terminal inside the theme folder and then enter the following command

    wget -r -nH -P docs -E -T 2 -np -k (your localhost URL)

    it should look something like this ->

    Once you run this, your html files should be inside your docs folder!

    Step 4: Push to GitHub

    So here's what we have now.

    -We got a domain name from namecheap + github student pack.
    -We used a website making tool to help up avoid writing code (GHOST).
    -We made the website but the https://localhost website that we run is offline, i.e. only we can see it because it's locally hosted.
    -We want someone to host it online 24/7 (for which we usually pay) but GitHub was like hey you can host a static website here for free no problem.
    -So we converted our handlebar .hbs (theme) files to .html for GitHub using wget.
    -Now we send all of this to the GitHub repository which is integrated with our domain name and Github takes care of the rest.

    Now sending files to GitHub is a process that you need to do EVERY TIME you make a new post. Like I'll have to do after I complete writing this. So what I personally suggest is you get familiar with github instead of copy pasting the commands every time, because it gets messy at times. You just need to learn the basics of add, commit, push, log, status, pull and branch.

    Step 5: The Last (finally)

    We've to get the
    SSL certificate : An SSL certificate is a digital certificate that authenticates a website's identity and enables an encrypted connection
    DNS : special Domain Name System (DNS) record that allows domain owners to specify who is allowed to issue SSL certificates for their domains
    And then move our websites to https so it's all safe and secure.

    Step 6: Editing and adding New Posts

    Technically not a step but eh. So to edit what you have to do is :

    1) Open ghost CLI using the commands
    2) Use ghost to create a new post/edit posts
    3) Download the theme again
    4) Convert to .html using wget
    5) Push to GitHub

    Seems overwhelming I know. But once you do it once or twice then you'll barely take 10 mins after writing a new post to deploy it. It's a valuable skill to learn and definitely worth the time.
